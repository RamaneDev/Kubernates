creating a StatefullSet stapes:
    - create a disk in used cluster by commandLine ( GKE): 
       gcloud compute disks create pv-a --zone=europe-west4-a --size=10GB
       gcloud compute disks create pv-b --zone=europe-west4-a --size=10GB
       gcloud compute disks create pv-c --zone=europe-west4-a --size=10GB
    
    - create a PV (persistent-volumes pv-a, pv-b, pv-c) using yaml file 
      see persistent-volumes-gcepd.yaml file in Git repo.

    - create a governing service (headless to use in StatefullSet object)
      see statefullset-kubia-headless.yaml

    - create a StatefullSet object 
      see kubia-statefullset.yaml


after that we can hit not by service headless (because its a cluster service) but trought Server API by proxy

after running the : kubectl proxy commandLine, we can use the commande below :

curl localhost:8001/api/v1/namespaces/default/pods/kubia-1/proxy/

the following request will post a data to a pod : 

curl -X POST -d "Hey there! This greeting was submitted to kubia-0." localhost:8001/api/v1/namespaces/default/pods/kubia-0/proxy/
